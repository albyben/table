在不同场景，用户对产品的功能和体验诉求都不相同。所以我们对应设定了不同布局模式，来应对不同需求场景。

## 通用顶栏
通用顶栏是在产品中任何页面都存在的，针对全局进行操作；

通用顶栏为固定内容，在整个产品上适用；

<img src="http://ikd.kingdee.com/kdrive/user/file?file_id=125278746" width="1096" height="52">

全局菜单--可以在任何页面唤出的常用功能级菜单，满足用户在不同业务中快速跳转的场景；

应用页签--固定应用和已经打开的临时应用；

应用中心--查看全部应用；

信息区--用户个人信息入口

## 首页布局
首页是用户进入产品时的第一个页面，它主要承载整体业务关键信息进行查看，能够掌握全局。系统需要主动引导用户开展业务，而不是让用户寻找。

首页的布局框架是固定的，如需要对首页进行定制，可能需要进行二次开发；

卡片区域可以通过对栅格的调整来改变卡片的布局策略；

首页的卡片有默认示例，用户可以通过页面编辑自定义所有卡片的内容及其排序；

典型页面：金蝶生态首页

<img src="http://ikd.kingdee.com/kdrive/user/file?file_id=125278870" width="1196" height="606">

页面编辑--自定义首页卡片

侧边工具栏--系统级服务入口

卡片区--承载自定义卡片的区域，卡片的尺寸和位置由用户控制（但会锁定在相应的栅格之上）

卡片布局调整

如果在首页进行卡片布局时，觉得卡片尺寸难以控制到自己想要的尺寸，可以通过BOS中属性修改来进行控制

## 工作台布局
当用户进入具体应用工作台时，他的关注点已经聚焦了具体应用，这个时候我们应该提供当前业务更细致的数据，和具体的单据操作功能。

工作台布局默认由业务侧配置，但是用户可以根据自己的习惯和需求进行定制。

典型页面：总账工作台、供应商工作台、B2B电商中心工作台、开发平台

## 录入/详情布局
录入/详情页的主要场景是对单一单据的录入、查看和处理。

一般有两种形态，单列布局和横向分割布局；

典型页面：要货申请单、任务审批

内容区

通用工具栏--页面级操作栏，在详情页仅包含按钮区域

锚点--用于页面内容的快速定位

内容面板--承载内容的面板，一般是用BOS中的高级面板/flex面板/分割容器

在BOS内容面板的选择如果需要考虑字段根据屏幕自适应，请不要使用flex面板

## 内容弹窗布局
BOS -> 容器 -> 样式 -> 容器快速布局

内容弹窗一般针对局部内容|少量内容的录入和查看。

我们分为小中大三个尺寸；

典型页面：新建应用、新建数据中心
<img src="http://ikd.kingdee.com/kdrive/user/file?file_id=125280080" width="978" height="598">


常规弹窗

<img src="http://ikd.kingdee.com/kdrive/user/file?file_id=125280193" width="1038" height="590">

页面型弹窗

标题区--告知用户当前位置

常规操作--一般只有关闭，特殊情况可添加最小化|全屏

内容区--承载内容的面板，在弹窗中一般是单列或双列

核心操作区--一般左侧为取消，右侧为核心操作按钮。特殊情况下也可以变成放置三个按钮（最多三个），但是有色彩指引的按钮只能有一个。

注意：弹窗会打断用户，不可在一个页面大量使用弹窗进行查看和录入。

1. 容器布局模式

2. 自身适配方式

样式调整请在BOS中设置：BOS -> 容器 -> 样式 -> 容器快速布局
